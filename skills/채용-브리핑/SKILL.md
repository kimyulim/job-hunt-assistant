# 채용 브리핑

## 실행 시점
사용자가 "오늘 브리핑 보여줘" 또는 "신규 공고 뭐 있어?"라고 요청할 때 실행.

## 절차

### 1. 트래커 전체 읽기
`01-Projects/채용-현황/지원-트래커.md` 파일 전체를 읽는다.

### 2. 오늘 추가된 행만 선별
- 트래커에 "추가일" 컬럼이 있다면 오늘 날짜인 행만 선별
- 추가일 컬럼이 없다면, 마지막 대화(직전 공고-수집 실행)에서 새로 추가한 행 기준으로 선별
- 애매하면 사용자에게 "최근 며칠치를 볼까요?" 물어보지 말고, 기본값으로 최근 실행 1회분 기준으로 보여줌

### 3. 마감 임박 여부 계산
- 마감일이 오늘 기준 5일 이내면 ⚠️ 표시
- 마감일이 "상시"인 경우 표시 없음

### 4. 출력 형식 (터미널 가독성을 위해 반드시 압축된 형태로 — 한 공고당 최대 2줄)
```
🆕 신규 공고 N건 (⚠️표시 = 확인 필요)

1. {회사명}({회사규모}) · {직무} · {플랫폼} · 마감 {마감일}
2. {회사명}({회사규모}) · {직무} · {플랫폼} · 마감 {마감일} ⚠️마감임박
3. {회사명}({회사규모}) · {직무} · {플랫폼} · 마감 {마감일} ⚠️경력재확인
...

📊 미지원 {N} · 지원완료 {N} · 보류 {N}  |  대기업{N} 중견{N} 중소{N} 스타트업{N} 미상{N}
```

### 절대 하지 말 것 (가독성 저해 요소)
- "매칭사유" 문장을 브리핑 본문에 풀어서 쓰지 않는다 (직무/지역/경력조건은 이미 트래커 필터링 통과한 것들이므로 반복 설명 불필요 — 트래커 원본에는 남겨두되 브리핑 출력에서는 생략)
- 근무지, 경력조건을 별도 줄에 나열하지 않는다 (이미 필터링을 통과했으므로 굳이 안 보여줘도 됨, 궁금하면 사용자가 트래커 원본이나 URL에서 확인)
- 한 공고당 3줄 이상 쓰지 않는다
- 이모지·화살표(→) 남발하지 않는다 — ⚠️ 표시만 사용

## 하지 말 것
- 자소서 초안, 자소서 작성 제안, "자소서부터 시작할게요" 같은 안내는 하지 않는다
- 브리핑은 공고 목록과 현황 요약까지만 하고 끝낸다. 지원 여부·자소서 작성은 사용자가 별도로 직접 판단/진행하는 영역이므로 관여하지 않는다

## 주의사항
- 이미 지원완료/제외 처리된 공고는 브리핑에 다시 띄우지 않음
- 트래커 파일을 요약이나 일부가 아니라 전체를 읽어서 정확한 현황(건수)을 계산할 것
- 상세 정보(매칭사유, 근무지, 경력조건)가 궁금하면 "{번호}번 자세히 보여줘"라고 요청하도록 안내 가능 (먼저 나서서 보여주지 않음)
