# 공고 수집

## 실행 시점
사용자가 "오늘 공고 확인해줘" 또는 "공고 수집해줘"라고 요청할 때 실행.
(추후 자동화 시 매일 아침 실행 예정)

## 절차

### 1. 사람인 메일 읽기
- Gmail에서 발신자에 "사람인" 또는 "saramin"이 포함된 최근 메일 검색 (최근 1~2일 이내)
- 메일 본문에서 공고 제목, 회사명, 공고 링크(URL) 추출

### 2. 각 공고 상세 확인 (중요 — 절대 생략 금지)
- 1번에서 추출한 각 링크를 WebFetch로 직접 열어서 확인:
  - 근무지 (메일 제목만으로 지역 판단 금지 — 반드시 상세페이지에서 확인)
  - 경력 조건
  - 코딩테스트 유무 (성격은 `채용-필터-기준/SKILL.md`의 "코테 판단 기준" 참고)
  - 마감일
  - **회사규모** (대기업 / 중견기업 / 중소기업 / 스타트업 — 아래 "회사규모 확인 방법" 참고)
- 정적 WebFetch로 상세정보(특히 근무지)가 안 나오는 경우(JS 렌더링 페이지):
  - 사람인은 `view-detail` URL 형식으로 재요청 시도
  - 그래도 안 되면 claude-in-chrome 스킬로 실제 브라우저를 열어서 확인
- 이 단계를 건너뛰고 메일 제목만으로 판단하지 말 것

### 3. 잡코리아 / 원티드 / 리멤버 웹검색
- 웹검색으로 아래 조건에 맞는 공고 탐색:
  - "잡코리아 UX UI 디자이너 신입 서울"
  - "원티드 프로덕트 디자이너 신입 서울 판교"
  - "리멤버 UX UI 디자이너 신입 서울" (리멤버는 커리어 SNS 채용공고 섹션 기준)
- 검색 결과에서 공고 제목, 회사명, 링크, 마감일 추출
- 가능하면 링크를 WebFetch로 열어 상세 조건 재확인
- 리멤버는 로그인 후에만 상세가 보이는 경우가 있음 → 안 열리면 제목/요약 정보만으로 1차 판단하고 "⚠️ 상세확인 필요" 표시

### 회사규모 확인 방법
- 사람인/잡코리아 상세페이지에는 보통 "기업형태: 대기업/중견기업/중소기업" 정보가 명시되어 있음 → 있으면 그대로 사용
- 원티드/리멤버처럼 기업형태가 안 나오는 경우: 회사명으로 웹검색 1회 (예: "{회사명} 기업규모") 해서 확인
- 그래도 불명확하면 "미상"으로 표기 (억지로 추측해서 잘못된 값 넣지 말 것)
- 스타트업 여부는 "설립연차 짧음 + 투자단계 언급" 등으로 판단, 애매하면 "중소기업"으로 분류

### 4. 필터링
- `skills/채용-필터-기준/SKILL.md`의 기준 적용:
  - 근무지: 서울 OR 판교 통과 / 그 외 지역 제외
  - 경력: 신입~3년차 통과 / 4년차 이상 제외, 상한 불명확하면 ⚠️ 표시하고 포함
  - 코테: "코테 판단 기준"에 따라 개발직군 알고리즘 테스트만 제외, 디자이너 대상 AI툴/실무 테스트는 포함(가산점)
  - 인턴은 기본 제외 (단, 대기업 채용전환형은 예외)

### 5. 중복 제거 (트래커 전체 스캔 필수)
- `01-Projects/채용-현황/지원-트래커.md` 파일 전체를 직접 읽어서 회사명 목록 확인
- 이미 트래커에 있는 회사명과 겹치면 제외 (요약이나 일부만 보지 말고 파일 전체 스캔)
- 여러 플랫폼(사람인/잡코리아/원티드/리멤버)에서 같은 회사+같은 직무가 중복 발견되면 하나로 합치고 "플랫폼" 칸에 발견된 곳 전부 표기

### 6. 트래커에 신규 공고 추가
필터링·중복제거 통과한 공고만 트래커에 새 행으로 추가:
| 회사명 | 회사규모 | 직무 | 플랫폼 | 마감일 | URL | 상태 | 매칭사유 |
- 상태는 항상 "미지원"으로 시작
- 매칭사유는 `채용-필터-기준/SKILL.md`의 "매칭 사유 작성 규칙" 형식 따름

### 7. 결과 보고
아래 형식으로 사용자에게 요약 출력:
```
🆕 신규 공고 N건

1. {회사명} ({회사규모}) - {직무} ({플랫폼})
   마감: {마감일} | {지역} | {경력조건}
   → {매칭사유}
```
마감이 5일 이내면 ⚠️ 표시 추가.

## 주의사항
- 지역 불명확한 공고는 절대 통과시키지 말고 반드시 WebFetch(또는 claude-in-chrome)로 확인
- 트래커 중복 제거는 파서/요약 신뢰하지 말고 원본 파일 전체 스캔
- 회사규모는 확실하지 않으면 "미상"으로 두고, 추측으로 단정짓지 말 것
